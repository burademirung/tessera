# ADR-0010: Static-First Astro + Capability-Gated R3F Island; Accessible SVG Graph Triples as A11y / Reduced-Motion / Low-End Fallback

**Status:** Accepted

---

## Context

Tessera requires a premium reference website that visualizes the live identity flow graph driven by real telemetry. The site must achieve Lighthouse performance score ≥ 95, WCAG 2.2 AA accessibility, and function on low-end devices and for users with motion preferences — while delivering a 3D WebGL experience for capable devices. Three concerns must be balanced: performance, accessibility, and progressive enhancement.

Research brief 09 (`docs/superpowers/research/09-frontend-3d-r3f-design.md`) established the following findings:

**Framework: Astro**
- Astro produces zero client-side JavaScript by default; components outside of islands are server-rendered to static HTML.
- The `client:only="react"` directive skips SSR and hydrates the component client-side only — required for WebGL/Canvas which needs browser APIs. However, `client:only` hydrates **immediately** on mount, not lazily — the component code must implement its own deferral via `IntersectionObserver` and capability checks.
- `slot="fallback"` on the island renders a placeholder while hydration is pending; a `priority` `<Image>` here becomes the LCP element.
- `<ClientRouter>` (View Transitions) + `transition:persist` allows navigation without tearing down the active WebGL context or SSE connection.

**Performance: R3F best practices**
- `frameloop="demand"` + `invalidate()`: render only when there is something to show; `invalidate()` requests a single frame (not a continuous loop), preventing idle GPU work.
- All nodes and edges rendered via `drei`'s `<Instances>` (one draw call for many objects) with shared `useMemo` geometry and material (`dispose={null}` to prevent premature disposal of shared resources).
- Edge-pulse animation implemented as a **shader uniform mutated in `useFrame`** — no React `setState`, no re-renders on the animation hot path.
- `React.lazy` + `<Suspense>` to split Three.js out of the initial JS bundle.
- `<Preload all>`, `<Bvh>` (fast pointer intersection), `<PerformanceMonitor>` + `<AdaptiveDpr>`, `dpr={[1,2]}` (clamp pixel ratio to device capability).

**Canvas is NOT an LCP candidate** (MDN, research brief 09 §6): the browser does not consider `<canvas>` content for Largest Contentful Paint. The LCP element must be the poster `<img>` rendered in `slot="fallback"`. The canvas bounding box must be reserved via `aspect-ratio` CSS to prevent Cumulative Layout Shift (CLS > 0 when the canvas appears).

**Accessibility: WCAG 2.2 AA**
- `<canvas>` is invisible to assistive technology — `aria-hidden="true"` on the decorative duplicate canvas, `role="img"` + `aria-label` on the accessible wrapper.
- A fully semantic SVG/HTML graph with keyboard navigation and ARIA must exist as a real DOM element — not hidden, not generated by canvas.
- Per WCAG 1.4.1: node types must be distinguished by icon + label, not color alone.
- Per WCAG 2.2.2: animation playing > 5 s must be pausable — a visible Pause button.
- Per WCAG 2.3.1: animation must not exceed 3 flashes per second.
- Per WCAG 1.4.3: text contrast ≥ 4.5:1 — critical on a light (`#FAFAFB`) background.

**The triple-duty fallback insight:**
Research brief 09 (§4, §7) identifies a significant architectural economy: the semantic SVG/HTML graph built for accessibility can simultaneously serve as:
1. The **a11y equivalent** (keyboard-navigable, ARIA, data table, non-canvas presentation of all graph data).
2. The **`prefers-reduced-motion` alternative** (no animation, no WebGL, no flicker).
3. The **low-end device fallback** (no GPU, no WebGL context, `deviceMemory < 4 GB`, `hardwareConcurrency < 4`).

Building one artifact for three purposes reduces maintenance burden and ensures they stay in sync. The SSE event feed wires to this SVG graph directly (for live data) even without WebGL.

**Fallback ladder** (`@pmndrs/detect-gpu`):
- GPU tier 3 → full WebGL, `dpr [1, 2]`.
- GPU tier 2 → reduced complexity, `dpr 1.5`.
- GPU tier 0–1 / no WebGL / `deviceMemory < 4 GB` → SVG graph (SSE-fed).
- `prefers-reduced-motion` / `Save-Data` / WebGL context loss → static poster.

## Decision

The Tessera site is built with **Astro (static-first, no adapter for marketing/content)** and one `client:only="react"` island for the identity graph. The island implements a **capability-gated `<Canvas>` mount** backed by a **semantic SVG/HTML graph** that serves as the accessible equivalent, reduced-motion alternative, and low-end fallback — a single artifact fulfilling all three roles.

**Astro configuration:**
- All marketing and content pages: zero client JS (static Astro + Markdown/MDX).
- Graph island: `<IdentityGraph client:only="react" transition:persist>` with `<ClientRouter>` for navigation.
- Fallback slot: `<Image slot="fallback" src={poster} priority loading="eager" fetchpriority="high" alt="Identity flow diagram">` — this is the LCP element.
- Canvas box reserved with `aspect-ratio: 16/9` CSS before hydration — CLS = 0.

**Capability-gate inside the island (executed client-side after hydration):**
1. Check `window.matchMedia("(prefers-reduced-motion: reduce)")`.
2. Check `navigator.connection?.saveData`.
3. Check `detect-gpu` tier and `navigator.deviceMemory` / `navigator.hardwareConcurrency`.
4. Attempt WebGL context on a hidden canvas element.
5. Decision tree:
   - Reduced-motion OR Save-Data → render SVG graph (static poster behind, no SSE animation).
   - GPU tier 0–1 OR no WebGL OR low device memory → render SVG graph (SSE-fed live data).
   - GPU tier 2 → `dpr [1, 1.5]` reduced R3F.
   - GPU tier 3 → full R3F `dpr [1, 2]`.

**SVG/HTML graph (always present in DOM):**
- Fully keyboard-navigable (tab order, enter/space for node details).
- ARIA: `role="img"` on the container, per-node `aria-label`, `aria-describedby` linking to a data `<table>`.
- Node types: distinguished by SVG icon + text label (not color alone).
- Animation: none by default; SSE events update node attributes (class, `data-status`) — CSS-only transitions, respect `prefers-reduced-motion`.
- Visible Pause button when animation is active.
- Pulse rate ≤ 3/s enforced in the SSE consumer.
- The canvas R3F view uses `role="img"` + `aria-hidden` (decorative duplicate); the SVG graph is the AT-visible representation.

**R3F performance (when WebGL path active):**
- `frameloop="demand"` + `invalidate()` on SSE pulse events.
- All nodes + edges: `<Instances>` shared `useMemo` geometry/material.
- Edge pulse: shader uniform in `useFrame`, no React state updates per frame.
- Lazy: `React.lazy(() => import("./Graph"))` + `<Suspense>`.
- Adaptive: `<PerformanceMonitor onDecline={() => setDpr(Math.max(dpr - 0.5, 1))} />` + `<AdaptiveDpr pixelated>`.
- SSE → zustand store (transient subscriber, no `setState`) → `useFrame` reads ref → `damp3` toward target.

**Design tokens:**
- Background: `#FAFAFB` (off-white).
- Text: `#1A1A1F` (charcoal, satisfies 4.5:1 contrast on `#FAFAFB`).
- Accent: one color, used exclusively on live flowing edges + primary CTA — preserves signal value.
- Single variable font, 8-pt scale, generous whitespace.
- Motion: `ease-out` ~240 ms for UI transitions; WebGL damp uses `maath`'s `damp3` (refresh-rate independent).

## Consequences

**Positive:**
- Lighthouse ≥ 95 is achievable with static Astro + lazy R3F + poster LCP element + reserved canvas box.
- WCAG 2.2 AA is satisfied by the always-present semantic SVG/HTML graph.
- One SVG artifact covers accessibility, reduced-motion, and low-end devices — minimal maintenance divergence.
- `transition:persist` keeps WebGL context and SSE connection alive across navigations.
- `frameloop="demand"` means zero GPU work when no events are arriving — no idle battery drain.
- `<Instances>` + shader uniforms allow hundreds of graph nodes in a single draw call.

**Negative / Tradeoffs:**
- `client:only="react"` hydrates immediately on mount — the IntersectionObserver gate must be implemented inside the component, not delegated to Astro's `client:visible` directive (which does not skip SSR).
- Two parallel graph representations (SVG + R3F) must be kept synchronized — a structural graph change (new node type) requires updating both.
- `detect-gpu` adds a small script to the island; it must not block LCP.
- WebGL context loss requires a recovery handler (`onContextLost`, fall back to SVG).
- SSE `EventSource` auto-reconnects on failure but does not buffer events — brief disconnections may cause missed pulses (acceptable for live visualization; the SVG graph always shows current state from the last received event).

---

## Alternatives Considered

| Option | Reason Rejected |
|---|---|
| Next.js (SSR/SSG) | Heavier framework overhead; Cloudflare Pages adapter for SSR adds complexity; Astro is purpose-built for content-heavy static sites with targeted hydration. |
| Pure WebGL (no R3F) | React Three Fiber's declarative API and drei utilities (Instances, PerformanceMonitor, AdaptiveDpr, Bvh) reduce R3F setup to ~20% of the raw Three.js equivalent; no compelling reason to forgo them. |
| CSS 2D animation fallback only (no SVG graph) | Would not satisfy WCAG 2.2 AA keyboard navigation or provide real-time SSE data to non-WebGL users. |
| Separate accessible page (e.g., `/graph-text`) | Fails WCAG 1.1.1 (the non-text content alternative must be in context, not linked away). Splits maintenance. |
| Canvas with ARIA only (no SVG) | Canvas interior is not traversable by assistive technology — `role="img"` + `aria-label` provides a description but not interactive exploration of individual nodes. |
